## 组件说明

### 1. 前置程序（Frontend Program）

**职责**：将人类/LLM 的意图转化为规范的 YAML 指令集

**子组件**：
- **语言检测器** — 自动或显式判断用户使用的自然语言
- **翻译层** — 将自然语言关键词映射到规范英文关键词
- **KV 解析器** — 解析 KV Line Format 为 AST
- **验证器** — 用 JSON Schema 校验 AST
- **编译器** — AST → YAML（带注释的内部格式）

**部署形态**：可以是一个独立的 CLI 工具，也可以是 ws-tasks 的内置模块

### 2. ws-tasks 守护进程

**职责**：执行指令集中的步骤

**特点**：
- **DAG 调度** — 自动识别步骤依赖关系，并行执行独立分支
- **能力调用器** — 加载和执行 `.cap.yaml` 描述的能力二进制
- **状态管理** — 追踪每个步骤和整个任务的执行状态
- **错误处理** — 支持 retry/ignore/stop/fallback 策略

### 3. 能力注册表（Capability Registry）

**职责**：管理所有可用的能力

**能力清单**：
| 能力 | 功能 | 状态 |
|------|------|------|
| ws-voice | 语音识别/合成 | 🚧 |
| ws-image | 图片处理 | 🚧 |
| ws-storage | 文件存储 | 🚧 |
| ws-db-query | 数据库查询 | 🚧 |
| ws-summarize | 文本摘要 | 🚧 |
| ws-extract | 信息提取 | 🚧 |
| ws-chart | 图表生成 | 🚧 |
| ws-email | 邮件发送 | 🚧 |
| ws-wechat-send | 微信通知 | 🚧 |
| ws-calc | 数据计算 | 🚧 |

### 4. 能力描述文件（.cap.yaml）

每个能力有一个自描述清单：

```yaml
name: ws-voice
version: 1.0.0
description: 语音识别与合成
binary: /usr/local/bin/ws-voice
args:
  model:
    type: string
    default: whisper-large-v3
  language:
    type: string
    default: zh
  diarize:
    type: boolean
    default: false
inputs:
  - type: file
  - type: ref
outputs:
  transcript: stdout.text
```

## 数据流示例

用户说：**"把这段会议录音转成文字，提取待办事项，然后发到群里"**

```
1. LLM → KV Line Format
   step: 转录 { cap: ws-voice, input: meeting.mp3 }
   step: 提取 { cap: ws-extract, input: $转录 }
   step: 通知 { cap: ws-wechat, input: $提取 }

2. 前置程序 → 翻译 + 解析 + 验证 + 编译
   → YAML (internal)

3. ws-tasks → DAG 调度
   → 转录(0s) → 提取(并行) → 通知(最后)
   → 各步骤调用对应能力二进制
   → 输出结果
```
